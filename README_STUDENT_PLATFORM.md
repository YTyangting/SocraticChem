# 学生交互式化学实验平台

## 概述

这是一个基于原有化学实验模拟框架改造的学生交互平台。原本的AI代理自动执行被替换为真实学生手动输入，保留了完整的物理引擎和反应系统。

## 文件说明

1. **`student_interactive_platform_simple.py`** - 主平台文件（推荐使用）
   - 简化设计，减少bug
   - 支持自然语言和XDL命令
   - 实时状态显示
   - 自动记录操作

2. **`test_platform.py`** - 测试脚本
   - 验证平台功能是否正常

3. **`student_records_*.jsonl`** - 实验记录文件
   - 自动生成，记录所有操作

## 快速开始

### 1. 启动平台
```bash
cd /home/yjh/socChem_final
python student_interactive_platform_simple.py
```

### 2. 基本操作

平台启动后，你可以：

1. **输入自然语言命令**：
   ```
   > 向试管1加入水10ml
   > 加热试管1
   > 将试管夹连接到铁架台
   > 从烧杯1倒入试管1
   > 等待5秒
   > 搅拌试管1
   ```

2. **输入XDL格式命令**：
   ```
   > {"action":"Add","vessel":"tube1","reagent":"water","volume":"10ml"}
   > {"action":"Heat","vessel":"tube1","temperature":"100"}
   ```

3. **使用系统命令**：
   ```
   > 状态      # 查看当前实验状态
   > 帮助      # 查看帮助信息
   > 历史      # 查看操作历史
   > 退出      # 结束实验
   ```

## 平台特点

### ✅ 已完成的功能

1. **状态显示**：
   - 显示所有器材的当前状态
   - 显示温度、内容物、体积
   - 显示连接关系
   - 显示操作计数

2. **命令解析**：
   - 自然语言解析（简化版）
   - XDL JSON格式解析
   - 错误提示和容错

3. **物理执行**：
   - 重用原有物理引擎
   - 支持所有基础操作（加料、加热、连接等）
   - 实时反馈执行结果

4. **实验记录**：
   - 自动保存所有操作到JSONL文件
   - 包含时间戳、命令、结果
   - 便于后续分析

### 🔧 技术实现

1. **最小化改动**：
   - 重用原有`ChemSimEngine`核心
   - 保持物理引擎完整性
   - 只替换输入输出层

2. **简化设计**：
   - 命令行界面，无需复杂依赖
   - 减少可能出错的组件
   - 清晰的错误提示

3. **兼容性**：
   - 与原有框架完全兼容
   - 使用相同的配置格式
   - 支持相同的操作类型

## 故障排除

### 常见问题

1. **导入错误**：
   ```
   ❌ 导入错误: No module named 'soc_chem_dia_refactored'
   ```
   **解决**：确保`soc_chem_dia_refactored.py`在同一目录下

2. **命令无法识别**：
   ```
   ❓ 无法识别，请用: 加入/加热/连接/倒入/等待/搅拌 或 JSON格式
   ```
   **解决**：使用更明确的表述或JSON格式

3. **执行失败**：
   ```
   ❌ 执行错误: 实验台上找不到器材 'xxx'
   ```
   **解决**：使用正确的器材ID（如tube1, beaker1等）

### 调试建议

1. 先运行测试脚本验证功能：
   ```bash
   python test_platform.py
   ```

2. 查看详细错误信息：
   ```python
   import traceback
   traceback.print_exc()
   ```

3. 检查记录文件了解操作历史
